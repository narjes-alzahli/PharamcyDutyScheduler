1. SYSTEM OVERVIEW
================================================================================

This is a comprehensive staff rostering system for pharmacy departments that:
- Generates monthly duty rosters automatically
- Balances workload fairly across staff
- Handles complex scheduling constraints
- Provides visual schedule displays
- Allows data management and customization

MAIN FEATURES:
✅ Automatic schedule generation using AI optimization
✅ Color-coded roster tables (like traditional pharmacy rosters)
✅ Employee workload analysis
✅ Leave and time-off management
✅ Skill-based assignment
✅ Fairness controls
✅ Visual schedule displays

3. COMPLETE FUNCTIONALITY GUIDE
================================================================================

📊 TAB 1: 👥 EMPLOYEES
=====================
WHAT IT DOES: Manage all staff information and capabilities

BUTTONS & FUNCTIONS:
• ➕ Add Employee: Creates new staff member with default settings
• Edit fields directly in the table

EDITABLE FIELDS:
• Employee Name: Staff member's full name
• Skills (Checkboxes):
  - Main Shift: Can work main pharmacy shifts
  - Outpatient: Can work outpatient department
  - Inpatient: Can work inpatient department  
  - Evening: Can work evening shifts (2:30-9:30 PM)
  - Night: Can work night shifts (9:30 PM-7 AM)
• Max Nights: Maximum night shifts per month (0-10)
• Max Evenings: Maximum evening shifts per month (0-10)
• Min Days Off: Minimum days off per month (1-10)
• Weight: Priority/fairness weight (0.1-10.0)
  - Higher weight = more assignments
  - Lower weight = fewer assignments
  - Set all to 1.0 for equal treatment

SUMMARY STATS DISPLAYED:
• Total Employees: Number of staff members
• Can Work Nights: How many can do night shifts
• Can Work Evenings: How many can do evening shifts
• Can Work All Shifts: How many are fully skilled

📋 TAB 2: DEMANDS
=================
WHAT IT DOES: Set daily staffing requirements for each day of the month

BUTTONS & FUNCTIONS:
• 🔄 Generate Month: Creates default demands for the selected month
• Edit daily requirements directly in the table

EDITABLE FIELDS (per day):
• Date: The specific date
• Main: How many main shift workers needed
• Outpatient: How many outpatient workers needed
• Inpatient: How many inpatient workers needed
• Evening: How many evening shift workers needed
• Night: How many night shift workers needed

DEFAULT VALUES:
• Main: 4 workers
• Outpatient: 4 workers
• Inpatient: 4 workers
• Evening: 2 workers
• Night: 2 workers

🏖️ TAB 3: TIME OFF
===================
WHAT IT DOES: Manage leave, time off, and special assignments

BUTTONS & FUNCTIONS:
• ➕ Add Leave: Assign leave to specific employees
• Edit existing time off entries

LEAVE TYPES AVAILABLE:
• DO: Day Off (regular day off)
• ML: Maternity Leave (maternity/paternity)
• W: Workshop (training/meetings)
• UL: Unpaid Leave (unpaid time off)
• H: Holiday (public holidays)
• STL: Study Leave (educational leave)

FORM FIELDS:
• Employee: Select from dropdown of all employees
• Date: Choose the date for time off
• Code: Select the type of leave

🔒 TAB 4: LOCKS
===============
WHAT IT DOES: Force or forbid specific assignments

BUTTONS & FUNCTIONS:
• ➕ Add Lock: Create assignment rules
• Edit existing locks

LOCK TYPES:
• Force (Must): Employee MUST work this shift on this date
• Forbid (Cannot): Employee CANNOT work this shift on this date

FORM FIELDS:
• Employee: Select from dropdown
• Date: Choose the date
• Shift: Select shift type (M, O, IP, A, N, DO, CL, ML, W, UL)
• Action: Choose "Force (Must)" or "Forbid (Cannot)"

⚙️ TAB 5: GENERATE
==================
WHAT IT DOES: Create the schedule with optimization settings

OPTIMIZATION SETTINGS:
• Time Limit: How long to search for solution (30-600 seconds)
  - 120 seconds is usually good
• Unfilled Coverage Penalty: How much to penalize missing coverage (100-10000)
  - Higher = prioritize coverage over fairness
  - 1000 is a good balance
• Fairness Weight: How much to prioritize fairness (0.0-50.0)
  - Higher = more equal distribution
  - 20.0-30.0 for very fair schedules
• Area Switching Penalty: Penalty for changing work areas (0.0-20.0)
  - Higher = prefer consistent assignments
  - 1.0 is usually good

BUTTONS & FUNCTIONS:
• 🚀 Generate Schedule: Creates the roster using current settings

RESULTS DISPLAYED:
• Success/Error messages
• Solve time (how long it took)
• Status (OPTIMAL, FEASIBLE, etc.)
• Total Assignments: Number of shift assignments
• Employees: Number of staff scheduled
• Days: Number of days in schedule
• Main Shifts: Number of main shift assignments

📅 TAB 6: VIEW SCHEDULE
=======================
WHAT IT DOES: Display the generated schedule

DISPLAY OPTIONS:
• Show Color-Coded Table: Main roster display with colors

FEATURES:

• Staff numbers (1, 2, 3, 4, 5...)
• Employee names
• Daily shift assignments with colors

• Shift legend with color meanings

4. FAIRNESS SYSTEM EXPLAINED
================================================================================

WHY SOME EMPLOYEES GET MORE SHIFTS THAN OTHERS:

The system balances multiple factors:
1. COVERAGE REQUIREMENTS: Must meet daily staffing needs
2. SKILLS: Only skilled employees can work certain shifts
3. CONSTRAINTS: Time off, locks, and rules limit assignments
4. FAIRNESS WEIGHT: Controls how much to prioritize equal distribution

EXAMPLE: Why Mona has 18 shifts but Hassan has 24:
• Hassan might be the only one who can work certain shifts
• Hassan might have more skills (can work more shift types)
• Mona might have more time off or restrictions
• The fairness weight might be too low

FAIRNESS FACTORS:
• Employee Weights: Higher weight = more assignments
• Skills: More skills = more possible assignments
• Leave: More leave = fewer assignments
• Special Requirements: Forced assignments increase workload
• Daily Requirements: Higher daily requirements = more total assignments

5. HOW TO MAKE SCHEDULES MORE FAIR
================================================================================

🎯 STEP-BY-STEP FAIRNESS GUIDE:

STEP 1: INCREASE FAIRNESS WEIGHT
• Go to Generate tab
• Increase "Fairness Weight" from 5.0 to 20.0-30.0
• This makes the system prioritize equal distribution

STEP 2: ADJUST EMPLOYEE WEIGHTS
• Go to Employees tab
• Set all "Weight" values to 1.0
• This ensures equal treatment

STEP 3: CHECK SKILLS DISTRIBUTION
• Make sure all employees have similar skills
• If Hassan can work all shifts but Mona can only work some, Hassan will get more assignments
• Consider training employees in more skills for better balance

STEP 4: REVIEW CONSTRAINTS
• Check Leave tab - maybe Mona has more days off
• Check Special Requirements tab - maybe Hassan is forced to work certain days
• Remove unnecessary constraints

STEP 5: ADJUST DEMANDS
• In Daily Requirements tab, reduce daily requirements if possible
• Lower demands = more flexibility = better fairness

🎯 RECOMMENDED SETTINGS FOR MAXIMUM FAIRNESS:
• Time Limit: 120 seconds
• Unfilled Coverage Penalty: 1000
• Fairness Weight: 25.0-30.0  ← MOST IMPORTANT
• Area Switching Penalty: 1.0

🔍 HOW TO CHECK FAIRNESS:
1. Generate a schedule
2. Go to View Schedule tab
3. Check "Show Employee Workload"
4. Look at the bar chart - bars should be similar heights
5. If not, increase Fairness Weight and regenerate

⚡ QUICK FIX FOR UNFAIR SCHEDULES:
1. Go to Generate tab
2. Set Fairness Weight to 25.0
3. Click Generate Schedule
4. Check the workload chart - should be more balanced

6. TROUBLESHOOTING
================================================================================

❌ COMMON ISSUES & SOLUTIONS:

ISSUE: "No data found for 2025-11"
SOLUTION: 
• Make sure you've generated a schedule first
• Check that the month/year selection matches the generated schedule
• Use the "Switch to [Generated Month]" button if available

ISSUE: "Failed to generate schedule"
SOLUTION:
• Reduce daily requirements in Daily Requirements tab
• Check that employees have the right skills
• Remove conflicting locks
• Increase time limit
• Lower fairness weight temporarily

ISSUE: "Employee workload shows 31 for everyone"
SOLUTION:
• This was a bug that's now fixed
• The chart now shows actual working shifts only
• Excludes day offs and leave

ISSUE: "Heatmap still visible"
SOLUTION:
• The heatmap option has been removed
• Only the color-coded table and workload chart are available
• This is intentional for a cleaner interface

ISSUE: "Table shows raw HTML"
SOLUTION:
• This has been fixed
• The table now displays properly with colors
• If you still see HTML, refresh the page

ISSUE: "Schedule not loading"
SOLUTION:
• Make sure the Streamlit app is running on port 8501
• Check the terminal for error messages
• Restart the app if needed
